# 机械臂使用教程

## Y1 机械臂初步运行与快速调试手册

### 🎛️ 机械臂核心调试流程（顺序启动 4 个终端）

请在 Ubuntu 系统中打开 **4 个独立的终端窗口**，严格按照以下顺序执行命令。

### 1️⃣ 终端 1：打通神经通路（启动 CAN 通信）

该终端作为守护进程，每秒会检查一次物理连接。**运行后请保持窗口开启，切勿关闭。**

```Plain Text
# 进入 CAN 脚本目录cd ~/y1_sdk_python/y1_ros2/can_scripts/

# 【注意】如果是这台电脑第一次连机械臂，需跑一次初始化（以后开机可跳过此步）
bash set_only_one_can.sh

# 真正合闸通电，激活并配置 can0 端口
bash start_can0.sh
```

### 2️⃣ 终端 2：给机械臂通电（编译并运行驱动）

启动机械臂的 ROS2 核心驱动。启动成功后，电机会正式上电，并发出微弱响声。

```Plain Text
cd ~/y1_sdk_python/y1_ros2/

# 编译驱动（仅在第一次使用、或修改了代码/参数后需要编译）
bash build.sh

# 刷新 ROS2 环境变量（【核心】任何新开窗口只要用机械臂，都必须先跑这行！）
source install/setup.bash

# 启动单臂控制模式
ros2 launch y1_controller single_arm_control.launch.py
```

### 3️⃣ 终端 3：开启相机眼睛（奥比中光驱动）

大模型与数据采集均依赖视觉流，在此将相机节点独立打开。

Bash

```Plain Text
cd ~/ros2_ws/
# 刷新相机空间环境
. ./install/setup.bash
# 启动相机广播图像话题，这里注意我们的相机型号
ros2 launch orbbec_camera dabai_dcw2.launch.py
```

### 4️⃣ 终端 4：运动控制与姿态微调

保持前 3 个终端常驻运行。在当前终端下发真正的动作控制指令。

- **测试 1：一键归零位置（电子编码器 0 点）**

- 使机械臂所有关节电机回到物理 `0` 点（此时机械臂将处于笔直挺立状态）。

```Plain Text
cd ~/y1_sdk_python/y1_ros2/
source install/setup.bash
bash control_scripts/go_zero_position.sh
```

- **测试 2：进入预备工作姿态（放倒低头）**

- 使机械臂从垂直状态弯下腰，趴在桌面上方，进入随时准备抓取的工作状态。

```Plain Text
source install/setup.bash
bash control_scripts/joint_position_control.sh
```

- **测试 3：水平面顺时针旋转 90 度（指向右侧 X 轴）**

- 机械臂在全零或趴着的状态下，如果朝向偏向 Y 轴，可通过控制 **1 号底座电机** 让其水平顺时针旋转 90 度。

- *根据右手法则，顺时针旋转 90 度对应的弧度值为 **`-1.57`**。*

```Plain Text
即把control_scripts/go_zero_position.sh文件里的
joint_position: [0, 0, 0, 0, 0, 0]这一行改成
joint_position: [-1.57, 0, 0, 0, 0, 0]在执行
bash control_scripts/go_zero_position.sh
```

- *\(注：具体话题名称可以先通过 **`ros2 topic list`** 进行核对确认。\)*

## 🛑 实验结束：安全断电与状态还原

为了保护电机及减速箱，实验完毕或需要重新调试时，切勿直接拔掉电源，请遵循以下复位下电步骤：

1. **安全归零**：在终端 4 运行 `bash control_scripts/go_zero_position.sh`，观察机械臂平稳站直。

2. **中止程序**：依次去往 终端 4、终端 3、终端 2、终端 1，按下 `Ctrl + C` 中止所有前台进程。终端 2 关闭时会听到电机“咔哒”一声，力矩完全卸载（机械臂完全放松）。

3. **强行清理残留**（防止有僵尸进程占用 CAN 通道或 ROS 端口）：

```Plain Text
sudo pkill -f y1_
sudo pkill -f single_arm_control
ros2 daemon stop
```

💡 **后续进阶**：真机本地控制绿灯通过后，即可开启虚拟环境 `conda activate imeta_data_collection` 运行数据采集脚本，或通过 `uv run` 联调 A100 服务器上的 OpenPI 策略进行分布式流式推理。
